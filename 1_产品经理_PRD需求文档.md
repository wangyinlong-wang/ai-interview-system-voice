## 2. 项目背景与目标

### 2.1 市场痛点分析

随着就业市场竞争日趋激烈，求职者在面试环节面临的挑战愈发严峻：

- **面试经验匮乏**：应届生每年超过1100万，其中约67%表示缺乏真实面试经验，首次面试紧张导致发挥失常的比例高达43%。
- **反馈渠道缺失**：传统模拟面试依赖亲友或导师，难以获得结构化、专业化的面试反馈，且受时间和人际关系约束。
- **准备效率低下**：求职者平均投递30+份简历才能获得1个面试机会，但多数人在面试前缺乏针对性的高效练习工具。
- **行业差异大**：不同岗位（技术/产品/运营/销售等）的面试考察点差异显著，通用型准备资料难以满足个性化需求。
- **AI技术成熟**：大语言模型（LLM）和多模态AI技术的快速发展，使得AI能够胜任结构化面试提问、实时评估和个性化反馈等复杂任务。

### 2.2 产品定位

> **AI 模拟面试系统** 是一款面向求职者的智能化面试练习平台，通过 AI 技术模拟真实面试场景，提供个性化的面试训练、实时的表现评估和结构化的改进建议，帮助用户系统性地提升面试能力和求职成功率。

- **产品 Slogan**：每一次练习，都是向 offer 更进一步。
- **核心价值主张**：随时随地，与 AI 面试官进行沉浸式模拟面试，获得专业级反馈，快速补齐短板。
- **目标市场**：初级职场求职者（0-3年经验）、转行/跳槽人员、在校大学生及研究生。
- **差异化优势**：基于简历内容的个性化提问、多维度智能评估体系、全链路面试训练闭环。

## 3. 用户画像

### 3.1 用户画像一：应届生 — 小王

| 维度 | 描述 |
|------|------|
| **基本信息** | 小王，22岁，计算机相关专业本科大四学生 |
| **核心诉求** | 希望获得人生第一份正式工作，对面试流程完全陌生，急需系统性练习 |
| **痛点** | 没有面试经验，一面对面试官就紧张；不清楚不同公司的面试风格差异；不知道自己的回答好不好 |
| **使用场景** | 秋招/春招前夕，每天晚上在宿舍用手机练习30分钟；面试前一天紧急抱佛脚 |
| **技术熟练度** | 高，习惯使用各类App和在线工具 |
| **付费意愿** | 中等，愿意为提升就业竞争力的工具付费，但对价格敏感 |
| **常用设备** | 手机（主要）、PC（偶尔） |

**用户故事：**

> 作为即将毕业的大学生，小王希望在正式面试前有一个安全、无压力的练习环境。他上传了自己的简历，选择了"Java后端开发"岗位和"初级"难度，开始了第一次AI模拟面试。AI面试官根据他的项目经历和求职意向，问了8个问题，涵盖自我介绍、项目深挖、八股文和场景题。面试结束后，他看到了自己的综合评分（72分）和能力雷达图，发现自己在"沟通表达"和"技术深度"两个维度较弱。他根据系统生成的改进建议，针对性地复习了相关知识点，并在3天后进行了第二次模拟面试，评分提升到了81分。最终，他在真实的面试中表现自信，顺利拿到了心仪的offer。

### 3.2 用户画像二：社招人员 — 李女士

| 维度 | 描述 |
|------|------|
| **基本信息** | 李女士，30岁，5年产品经理经验，希望跳槽到更大的平台 |
| **核心诉求** | 长时间没有面试，需要恢复面试状态；针对目标公司的岗位进行精准准备 |
| **痛点** | 过往面试经验局限于上一家公司，缺乏多样化的面试训练；对新兴面试题型（如案例分析、产品设计题）不熟悉；不确定自己的回答在市场中处于什么水平 |
| **使用场景** | 工作日晚间、周末，使用PC端进行深度练习；面试前1-2周集中训练 |
| **技术熟练度** | 高，熟练使用各类效率工具和SaaS产品 |
| **付费意愿** | 较高，将面试准备视为职业投资，愿意为高质量服务付费 |
| **常用设备** | PC（主要）、手机（碎片化时间） |

**用户故事：**

> 作为有5年经验的资深产品经理，李女士决定跳槽到一家互联网大厂。她已经有3年没有参加过正式面试，对面试流程感到生疏。她在AI模拟面试系统中上传了最新简历，选择了"高级产品经理"岗位和"高级"难度，并开启了"压力面试"模式。AI面试官基于她过往负责的核心项目，深入追问产品决策逻辑、数据指标、团队协作等维度的问题。面试结束后，她查看了详细的评估报告，发现自己在"逻辑思维"和"数据敏感度"方面表现优秀，但"抗压能力"和"创新思维"需要加强。她利用系统的"对比分析"功能，追踪了自己在一周内的4次模拟面试成绩变化趋势，并参照系统推荐的"优秀回答范例"优化了自己的回答策略。最终，她在真实的大厂面试中从容应对，成功拿到了P7级别的offer。

## 5. 核心用户流程

### 5.1 流程一：新用户首次使用（完整旅程）

```
[开始]
  |
  v
[进入首页/landing页]
  |   * 展示产品核心价值 proposition
  |   * 显眼的「开始模拟面试」CTA按钮
  |
  v
[注册/登录]
  |   * 支持手机号注册、微信授权登录
  |   * 新用户首次引导
  |
  v
[上传简历]
  |   * 拖拽或点击上传 PDF/DOC/TXT 文件
  |   * 实时展示上传进度
  |   * 解析完成后展示结构化预览
  |
  v
[确认/编辑简历信息]
  |   * 用户检查解析结果准确性
  |   * 支持手动修改和补充信息
  |   * 点击「确认并开始面试」
  |
  v
[面试准备页]
  |   * 选择目标岗位（下拉+搜索）
  |   * 选择面试难度（初级/中级/高级）
  |   * 选择面试形式和题数
  |   * 确认本次面试配置
  |
  v
[进入面试房间]
  |   * AI面试官开场白
  |   * 展示面试进度和计时
  |   * 用户逐题回答（文字/语音）
  |   * AI根据回答智能追问
  |   * 支持跳过和提前结束
  |
  v
[面试结束]
  |   * 用户确认结束或题目全部完成
  |   * 展示「评估生成中」loading态
  |
  v
[查看评估报告]
  |   * 综合评分展示
  |   * 能力雷达图
  |   * 逐题点评和改进建议
  |   * 引导用户开始下一次面试
  |
  v
[结束/分享]
```

## 7. 非功能需求

### 7.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 页面首屏加载时间 | ≤ 2秒 | 首页关键内容优先渲染 |
| 简历解析耗时 | ≤ 10秒（PDF≤5页） | 用户可接受的上限，需有进度提示 |
| AI首次响应时间 | ≤ 3秒 | 面试官首次回应的等待时间 |
| AI追问响应时间 | ≤ 5秒 | 追问问题的生成时间 |
| 评估报告生成时间 | ≤ 15秒 | 面试结束到报告展示的时间 |
| 并发用户数 | ≥ 500人同时在线 | MVP阶段目标 |
| 系统可用性 | ≥ 99.5% | 月度统计 |

### 7.2 安全需求

| 需求项 | 要求 |
|--------|------|
| 用户数据加密 | 用户密码采用bcrypt加密存储；敏感信息（手机号）加密存储 |
| 传输安全 | 全站HTTPS，TLS 1.2+ |
| 简历文件安全 | 简历文件加密存储，仅用户本人可访问 |
| 面试内容安全 | 面试问答记录加密存储，不用于模型训练（除非用户明确授权） |
| 隐私政策 | 明确的隐私政策告知用户数据使用方式 |
| 访问控制 | 严格的身份认证和权限校验，防止越权访问 |
| 防攻击 | 接口防SQL注入、XSS、CSRF攻击；登录防暴力破解 |

### 7.3 兼容性需求

| 平台 | 支持范围 |
|------|----------|
| PC端浏览器 | Chrome 90+、Edge 90+、Firefox 88+、Safari 14+ |
| 移动端浏览器 | iOS Safari 14+、Android Chrome 90+ |
| 响应式适配 | 适配 375px - 1920px 宽度范围 |
| 小程序 | 微信小程序（后续版本） |

### 7.4 可扩展性需求

| 需求项 | 说明 |
|--------|------|
| 多模型支持 | AI引擎层抽象封装，支持切换不同LLM（GPT/Claude/文心/通义等） |
| 岗位扩展 | 岗位题库采用模块化设计，支持快速新增岗位类型 |
| 多语言支持 | 系统架构预留国际化能力，支持后续扩展英文等语言版本 |
| 插件化评估 | 评估维度采用插件化设计，支持自定义新增评估维度 |
| 第三方集成 | 预留与招聘平台、在线教育平台的API集成能力 |

### 7.5 数据需求

| 需求项 | 说明 |
|--------|------|
| 数据备份 | 用户数据每日增量备份，保留30天 |
| 日志记录 | 关键操作日志保留90天，支持审计追溯 |
| 埋点方案 | 前端+后端全链路埋点，覆盖核心用户行为漏斗 |
| 数据看板 | 产品运营后台，实时监控关键业务指标 |

## 9. 风险与应对

| 风险类别 | 风险描述 | 影响程度 | 应对措施 |
|----------|----------|----------|----------|
| 技术风险 | 简历解析准确率不足 | 高 | 接入成熟的OCR+NLP服务，预留人工校对入口 |
| 技术风险 | AI响应延迟过高 | 高 | 使用流式输出，优化prompt减少token数；准备降级方案 |
| 产品风险 | AI提问质量不稳定 | 高 | 建立题库模板+AI生成的混合模式，关键问题预制 |
| 产品风险 | 评估标准主观性强 | 中 | 基于行业标准和真实面试数据校准评分模型 |
| 商业风险 | 用户付费意愿不足 | 中 | MVP阶段聚焦免费功能，通过用户反馈验证付费需求 |
| 运营风险 | 冷启动用户获取困难 | 中 | 与高校就业中心、求职社区合作推广 |
