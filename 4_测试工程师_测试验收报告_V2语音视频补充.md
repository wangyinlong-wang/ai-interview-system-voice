## 1. 测试概述

### 1.1 测试范围

本次补充测试覆盖 AI 模拟面试系统 V2 版本新增的两大核心功能模块：

| 模块类别 | 具体模块 | 测试重点 |
|----------|----------|----------|
| 语音面试 | 语音输入(STT)、语音合成(TTS)、语音队列、模式切换 | 识别准确率、播报流畅度、模式切换稳定性、浏览器兼容性 |
| 面对面面试 | 3D场景渲染、面试官模型、口型同步、表情动画、用户摄像头 | 场景加载性能、帧率达标、口型同步延迟、表情切换自然度 |
| 集成测试 | 准备页→面试房间→评估报告完整链路 | 多模式组合、数据一致性、降级方案有效性 |
| 兼容性 | 浏览器兼容性(Web Speech API / WebGL) | Chrome/Firefox/Safari/Edge/移动端 |
| 性能测试 | 3D加载时间、语音延迟、资源占用 | 首屏加载、语音同步延迟、内存/CPU/GPU占用 |
| 安全隐私 | 麦克风/摄像头数据安全 | 数据不上传、权限及时释放 |

| 摄像头数据泄露 | 用户视频画面被上传或录制 | 高 | 1. getUserMedia获取的流仅在本地展示 2. 不传输到任何服务器 3. 提供关闭摄像头的选项 | **低(已控制)** |
| 权限未释放 | 面试结束后麦克风/摄像头权限未释放 | 中 | 1. 组件卸载时停止所有MediaStream轨道 2. SpeechRecognition实例正确abort 3. speechSynthesis.cancel | **低(已控制)** |

### 8.4 降级方案有效性评估

| 降级场景 | 降级方案 | 有效性 | 建议 |
|----------|----------|--------|------|
| 浏览器不支持语音API | 自动降级为文字模式 | 有效 | 文字模式功能完整，用户体验无损失 |
| Firefox仅支持TTS | 隐藏麦克风，保留语音播报 | 有效 | 用户可听AI播报，但需手动输入 |
| 浏览器不支持WebGL | 显示降级页面，引导至文字面试 | 有效 | 降级页面清晰，跳转顺畅 |
| 设备性能不足 | 自动启用性能模式 | 基本有效 | 低端设备仍可能卡顿，需持续优化 |
| 麦克风权限被拒绝 | 显示提示，保留文字输入 | 有效 | 不影响面试进行 |
| 摄像头权限被拒绝 | 显示占位图，3D功能继续 | 有效 | 面试核心功能不受影响 |
