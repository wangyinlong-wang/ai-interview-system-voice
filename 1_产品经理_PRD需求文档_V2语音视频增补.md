## 新增模块一：语音面试模块

### 4.8 语音面试模块

#### 4.8.1 模块概述

语音面试模块是在现有文字面试基础上的重大体验升级，旨在为用户提供更接近真实面试场景的自然交互方式。用户可通过麦克风进行语音输入，AI面试官以语音合成方式进行播报，同时保留完整的文字记录，实现"语音为主、文字为辅"的双模式体验。

**模块定位**：
- 核心价值：降低输入门槛、提升沉浸感、模拟真实面试节奏
- 目标用户：习惯口头表达的用户、希望训练语言组织能力的用户、追求高沉浸感的用户
- 使用场景：PC端（推荐，配合耳机/麦克风）、移动端（方便随时随地练习）

**模块架构图**：

```
+-------------------------+  +-------------------------+
|      用户输入层          |  |      AI输出层           |
|  +-------------------+  |  |  +-------------------+  |
|  |   麦克风采集       |  |  |  |   AI语音合成TTS   |  |
|  |   （WebRTC）      |  |  |  |   （音色/语速）   |  |
|  +---------+---------+  |  |  +---------+---------+  |
|            |            |  |            |            |
|  +---------v---------+  |  |  +---------v---------+  |
|  |   语音识别ASR      |  |  |  |   语音播放器      |  |
|  |  （实时转写+纠错） |  |  |  |  （Web Audio API）|  |
|  +---------+---------+  |  |  +-------------------+  |
|            |            |  +-------------------------+
|  +---------v---------+  |
|  |   转写文字编辑区   |  |
|  |  （可编辑修正）    |  |
|  +-------------------+  |
+-------------------------+
            |
+-------------------------+
|      文字记录层         |
|  完整面试文字 transcript |
|  支持回看、导出         |
+-------------------------+
```

#### 4.8.2 功能需求清单

| 功能点 | 详细描述 | 优先级 | 验收标准 |
|--------|----------|--------|----------|
| 语音输入 | 用户按住麦克风按钮说话，松开结束采集；支持点击切换"按住说话"和"点击说话"两种模式 | P0 | 支持连续说话60秒不中断；PC端兼容耳机/外接麦克风；移动端兼容系统自带麦克风 |
| 语音转写显示 | 识别过程中实时显示转写文字，采用流式展示效果（文字逐字出现），识别完成后整句校正显示 | P0 | 转写延迟≤500ms（从说话结束到文字显示）；支持实时流式展示；支持在识别过程中看到中间结果 |
| AI语音播报 | AI面试官回答通过语音合成（TTS）播报，播报时面试官头像有音频波动画，播报完成后自动进入倾听状态 | P0 | 首包延迟≤2秒（从AI生成文字到开始播报）；播报流畅无卡顿；播报过程中可随时打断 |
| 文字+语音双模式 | 支持文字/语音随时切换，语音模式下仍显示完整文字记录；用户可在语音播报过程中查看文字内容 | P0 | 切换延迟≤300ms；文字记录完整不丢失；语音和文字内容一致 |
| 语音质量检测 | 面试开始前检测麦克风权限和环境噪音；面试过程中实时监测音量大小，音量过低或过高时给出提醒 | P1 | 权限检测在首次使用时自动触发；环境噪音检测给出"安静/正常/嘈杂"三级提示；音量提醒在状态栏显示 |
| 多音色选择 | 提供至少3种面试官音色：①男声沉稳（低沉、语速适中，适合技术岗）；②女声亲和（温柔、有亲和力，适合产品/运营岗）；③女声干练（清晰利落，适合管理岗） | P1 | 切换音色后即时生效；音色样本可在线试听；用户偏好设置保存在服务端 |
| 语速控制 | 用户可调AI播报语速，提供5档：0.5x（慢速）/ 0.75x（偏慢）/ 1x（正常）/ 1.25x（偏快）/ 1.5x（快速），默认1x | P1 | 切换语速即时生效；支持在面试过程中随时调整；语速调整后当前播报句即刻生效 |
| 语音识别优化 | 针对技术术语（如"Kubernetes"、"微服务"）、英文单词、专业名词进行识别优化；支持用户自定义词库 | P2 | 技术术语识别准确率≥85%；支持常见开发框架/工具的英文识别；用户可添加个人词库 |
| 播报打断机制 | 用户在AI播报过程中可随时点击"打断"按钮或开始说话，AI立即停止播报并切换为倾听状态 | P1 | 打断响应延迟≤200ms；打断后语音输入立即启用；打断操作有明确的视觉反馈 |
| 语音播放控制 | 支持暂停/继续/重听功能，用户对未听清的内容可点击重听 | P1 | 暂停后继续播放衔接自然；重听功能可定位到任意AI发言位置 |
| 静默检测 | 用户说话过程中出现3秒以上静默自动结束采集，避免长时间占用麦克风 | P1 | 静默时长阈值可配置（2-5秒可调）；静默结束后自动提交识别结果 |
| 面试模式切换入口 | 在面试准备页和面试房间页均提供文字/语音模式切换入口；首次进入语音模式时有操作引导 | P1 | 切换入口清晰可见；首次引导以浮层形式展示，用户可手动关闭 |

#### 4.8.3 交互流程详解

**语音输入流程**：

```
[用户按住麦克风按钮]
      |
      v
[采集开始] --> 麦克风图标变为"收集中"状态（红色呼吸灯动画）
      |
      v
[用户说话中] --> 实时展示音量波形动画 + 中间转写文字流式显示
      |
      v
[用户松开按钮 / 静默3秒自动结束]
      |
      v
[采集结束] --> 麦克风图标恢复常态，展示完整转写文字
      |
      v
[文字可编辑期] --> 用户有5秒时间修正识别结果
      |
      v
[自动提交 / 用户手动提交]
      |
      v
[AI处理] --> 展示"思考中"状态 --> AI文字回答 + 语音播报
```

**AI语音播报流程**：

```
[AI文字回答生成完成]
      |
      v
[语音合成请求] --> TTS引擎生成音频流
      |
      v
[首包音频就绪] --> 面试官头像开始音频波动画，状态切换为"发言中"
      |
      v
[语音播放中] --> 当前播报的文字句高亮显示，字幕同步滚动
      |
      v
[播报完成] --> 面试官头像恢复常态，状态切换为"倾听中"
      |
      v
[等待用户输入] --> 麦克风按钮闪烁提示用户可以说话
```

#### 4.8.4 异常处理

| 异常场景 | 处理方式 | 用户感知 |
|----------|----------|----------|
| 麦克风权限被拒绝 | 引导用户前往浏览器设置开启权限；提供文字模式作为备选 | 弹窗提示 + 一键跳转设置（移动端）/ 指引手动开启（PC端） |
| 语音识别失败/无结果 | 自动重试1次，仍失败则提示用户检查麦克风；保留文字输入作为备选 | Toast提示"未识别到语音，请重试或切换文字模式" |
| 语音识别结果偏差大 | 用户可在编辑期内手动修正；提供"重新说一遍"快捷操作 | 转写文字区域可直接点击编辑 |
| AI语音合成失败 | 自动降级为文字模式展示回答；记录错误日志 | 静默降级，不中断面试流程，文字正常展示 |
| 网络波动导致语音传输中断 | 本地缓存已采集的音频片段，恢复后自动续传 | 状态栏短暂提示"网络波动，正在恢复" |
| 环境噪音过大 | 实时检测并在状态栏提示"环境较嘈杂，建议使用耳机" | 状态栏黄色警告提示，不中断面试 |

#### 4.8.5 界面状态说明

| 状态 | 视觉表现 | 交互说明 |
|------|----------|----------|
| 等待输入 | 麦克风按钮呈灰色常亮，面试官状态为"倾听中" | 用户可按住说话或切换文字输入 |
| 采集中 | 麦克风按钮呈红色呼吸灯动画，显示音量波形 | 松手结束或静默3秒自动结束 |
| 识别中 | 麦克风按钮呈黄色旋转loading态，文字区域逐字显示 | 此时不可操作，等待识别完成 |
| 可编辑 | 转写文字高亮显示，底部出现"重新说"和"确认提交"按钮 | 用户有5秒时间修正文字 |
| AI思考中 | 面试官头像旁显示"思考中..."，麦克风禁用 | 等待AI生成回答 |
| AI播报中 | 面试官头像音频波动画，当前句文字高亮 | 用户可点击打断按钮或按住麦克风打断 |

## 新增模块二：面对面模拟面试模块

### 4.9 面对面模拟面试模块

#### 4.9.1 模块概述

面对面模拟面试模块是系统的沉浸式体验升级，通过Three.js构建3D虚拟面试官形象，结合语音合成与口型同步技术，为用户营造接近真实视频面试的场景感。用户可与3D虚拟面试官进行"面对面"交流，面试官具备逼真的表情、动作和口型动画，极大提升面试训练的真实感和代入感。

**模块定位**：
- 核心价值：打造最接近真实视频面试的训练环境，帮助用户克服镜头恐惧
- 目标用户：即将参加视频面试的用户、希望获得最高沉浸感的用户、需要练习镜头表达的用户
- 使用场景：PC端为主（推荐，需要较好显卡性能），支持Chrome/Edge/Firefox浏览器
- 技术基础：Three.js（3D渲染）+ Web Speech API（语音识别）+ TTS（语音合成）+ MediaDevices API（摄像头）

**模块整体架构**：

```
+---------------------------------------------------------------+
|                   面对面模拟面试系统                           |
+----------------+--------------+----------------+--------------+
|   3D渲染引擎   |   动画系统    |   语音系统      |   用户系统   |
|   (Three.js)   |   (Blend    |   (ASR+TTS)   |   (Media    |
|                |   Shapes)   |               |   Devices)  |
+----------------+--------------+----------------+--------------+
|                                                               |
|  +------------------+  +------------------+  +-------------+  |
|  |  虚拟面试官形象  |  |   3D虚拟场景     |  |  用户视频   |  |
|  |  - 形象选择      |  |   - 会议室/      |  |  小窗       |  |
|  |  - 口型同步      |  |     办公室/      |  +-------------+  |
|  |  - 表情动画      |  |     咖啡厅       |                 |
|  |  - 身体动作      |  |   - 动态光影     |  +-------------+  |
|  |  - 状态指示      |  |   - 背景音效     |  |  控制面板   |  |
|  +------------------+  +------------------+  +-------------+  |
|                                                               |
+---------------------------------------------------------------+
```

#### 4.9.2 功能需求清单

| 功能点 | 详细描述 | 优先级 | 验收标准 |
|--------|----------|--------|----------|
| 3D虚拟面试官 | 使用Three.js构建逼真的3D面试官形象，支持至少2种形象（男性/女性），模型细节包括：面部五官、头发、衣着、手势。面试官位于屏幕中央偏右位置，面向用户 | P0 | 模型加载完成≤3秒；支持鼠标拖拽360°观察面试官；模型面数优化，中端设备帧率≥30fps |
| 口型同步 | AI语音播报时，3D模型的嘴唇动作与语音精准同步。基于TTS音素数据驱动Blend Shape变形，实现自然开口、闭口、元音口型变化 | P0 | 口型延迟≤200ms；支持中文音节映射；播报过程中口型自然无突兀感 |
| 表情动画 | 面试官根据面试内容变化表情：①提问时-专注认真；②用户回答精彩时-微笑赞许；③思考时-微微皱眉/托腮；④鼓励时-点头微笑。表情通过面部Blend Shape实现平滑过渡 | P0 | 表情切换过渡时间300-500ms；至少实现5种基础表情；表情与语义匹配准确率≥70% |
| 用户视频画面 | 屏幕右下角显示用户摄像头画面（镜像显示），模拟真实视频面试布局。画面尺寸约240×180px，支持拖拽调整位置 | P0 | 视频延迟≤300ms；镜像显示正确；画面有圆角和轻微阴影；支持点击放大预览 |
| 面试官动作 | 面试官具备自然动作：①倾听时-身体微微前倾表示关注；②提问时-手势配合（伸手/点头）；③赞许时-竖起大拇指或点头；④记录时-手持虚拟笔做笔记动作。动作通过骨骼动画驱动 | P1 | 动作自然流畅，不僵硬；动作触发频率适中（每题1-2次）；动作与语义有一定关联 |
| 虚拟场景 | 3D会议室/办公室环境背景，支持至少2种场景切换：①标准会议室（深色木桌、皮质座椅、落地窗）；②现代办公室（白色简约、绿植装饰、玻璃隔断）。场景包含动态光影和背景环境音效 | P1 | 场景加载≤3秒；场景切换过渡动画≤1秒；光影效果真实自然 |
| 面试状态指示 | 面试官头顶上方显示当前状态标签，状态包括："倾听中"（绿色）/ "思考中"（黄色）/ "提问中"（蓝色）/ "评价中"（紫色）。状态变化有过渡动画 | P1 | 状态切换延迟≤300ms；状态标签与面试官实际行为一致；视觉上清晰可辨 |
| 开场/结束动画 | 面试开始时，面试官从座椅上起身迎接用户；面试结束时，面试官起身送别。包含握手示意、微笑点头等礼仪动作 | P2 | 开场动画时长5-8秒；结束动画时长3-5秒；动画播放流畅；用户可跳过动画 |
| 场景互动 | 用户可通过鼠标拖拽旋转视角观察面试官；鼠标悬停在面试官身上时，面试官目光跟随鼠标移动 | P2 | 视角旋转流畅60fps；目光跟随自然，有最大角度限制（不扭头超过45°） |
| 字幕/对话记录 | 屏幕底部显示对话字幕区域（可收起），展示当前轮次的问答文字；支持查看完整对话历史 | P0 | 字幕与语音同步延迟≤200ms；支持收起/展开；文字清晰可读，有半透明背景 |
| 面试官形象切换 | 支持在面试准备阶段选择不同面试官形象；形象选择页提供形象预览和简短性格描述 | P1 | 至少提供2种基础形象；切换后即时生效；形象偏好保存在用户设置中 |

#### 4.9.5 表情动画状态机

```
                    +-------------+
                    |   初始状态   |
                    |  (Idle等待) |
                    +------+------+
                           |
          +----------------+----------------+
          |                |                |
          v                v                v
   +-------------+  +-------------+  +-------------+
   |  提问状态   |  |  倾听状态   |  |  思考状态   |
   |  (专注认真) |  |  (前倾关注) |  |  (托腮沉思) |
   +------+------+  +------+------+  +------+------+
          |                |                |
          v                v                v
   +-------------+  +-------------+  +-------------+
   |  赞许状态   |  |  鼓励状态   |  |  评价状态   |
   |  (微笑点头) |  |  (手势引导) |  |  (记录笔记) |
   +-------------+  +-------------+  +-------------+
```

表情切换规则：
- AI播报开始时 → 切换至"提问状态"
- AI播报结束，等待用户回答 → 切换至"倾听状态"（身体前倾）
- 用户回答超过10秒 → 随机切换至"鼓励状态"（点头微笑）
- 检测到用户回答优秀关键词 → 切换至"赞许状态"（微笑+竖大拇指）
- AI生成回答前的思考间隙 → 切换至"思考状态"（托腮/眼神上移）
- 面试结束生成评价时 → 切换至"评价状态"（手持笔做记录状）

#### 4.9.6 异常处理

| 异常场景 | 处理方式 | 降级策略 |
|----------|----------|----------|
| WebGL不支持 | 检测浏览器是否支持WebGL，不支持则提示并引导使用语音模式 | 降级为语音面试模式（仅语音无3D） |
| 设备性能不足（帧率<20fps） | 自动降低3D模型精度、关闭光影效果、简化背景 | 提供"性能模式"开关，用户可手动降低画质 |
| 3D模型加载超时（>5秒） | 显示加载进度条，超时后提示用户检查网络 | 超时后自动降级为语音模式 |
| 摄像头权限被拒绝 | 面试官仍以3D形式呈现，但用户视频画面显示占位提示；引导用户开启权限 | 无用户视频画面，其余功能正常 |
| 摄像头 unavailable | 同上处理，提示"未检测到摄像头" | 同上 |
| 浏览器兼容性不足 | 进入前进行兼容性检测，不支持Three.js的浏览器拦截并推荐Chrome/Edge | 引导下载推荐浏览器 |

## 非功能需求补充

### 7.6 语音/视频相关性能需求

#### 7.6.1 语音处理性能

| 指标 | 要求 | 测试条件 | 说明 |
|------|------|----------|------|
| 语音识别延迟 | ≤ 500ms | 网络良好（延迟<50ms），标准普通话 | 从用户说话结束到转写文字完整显示的时间 |
| AI语音合成首包延迟 | ≤ 2秒 | TTS引擎预热完成，文本长度<200字 | 从AI文字回答生成完成到语音开始播放的时间 |
| AI语音合成完整延迟 | ≤ 5秒 | 文本长度500字以内 | 从文字生成完成到完整音频合成完成的时间 |
| 语音打断响应延迟 | ≤ 200ms | 用户开始说话到AI播报停止 | 包括语音活动检测(VAD)+指令响应时间 |
| 语音播放流畅度 | 无卡顿、无断音 | 网络带宽≥1Mbps | 播报过程中不允许出现可感知的停顿 |
| 语音同步误差 | ≤ 200ms | 字幕文字与语音播报同步 | 当前播报句的高亮与语音播放精准同步 |

#### 7.6.2 3D渲染性能

| 指标 | 要求 | 测试设备 | 说明 |
|------|------|----------|------|
| 3D场景首屏加载 | ≤ 3秒 | 100Mbps网络，标准设备 | 从页面加载到3D场景完全渲染并可交互 |
| 3D模型加载 | ≤ 2秒 | 同上 | 面试官模型从请求到加载完成 |
| 口型同步延迟 | ≤ 200ms | 同上 | TTS音素输出到3D口型渲染的延迟 |
| 帧率 | ≥ 30fps | 中端设备（i5/集成显卡） | 3D场景持续渲染帧率 |
| 高端设备帧率 | ≥ 60fps | 高端设备（i7/独显） | 开启全部特效时的帧率 |
| 内存占用 | ≤ 500MB | 中端设备 | 面对面面试页面内存占用峰值 |
| GPU占用 | ≤ 60% | 中端设备 | 避免影响其他系统功能 |

#### 7.6.3 摄像头与权限体验

| 需求项 | 要求 | 说明 |
|--------|------|------|
| 麦克风权限引导 | 首次进入语音/面对面模式时，友好引导开启麦克风权限 | 说明用途（"需要麦克风进行语音面试"），提供图文并茂的开启指引 |
| 摄像头权限引导 | 首次进入面对面模式时，友好引导开启摄像头权限 | 说明用途（"摄像头仅在本地展示，不会上传服务器"），消除隐私顾虑 |
| 权限拒绝处理 | 权限被拒绝后，提供备选方案（文字模式/语音无视频模式） | 不强制要求权限，确保用户可继续使用基础功能 |
| 权限状态记忆 | 记住用户上次的选择，下次进入时不再重复请求（除非用户重置） | 避免重复弹窗打扰用户 |
| 设备热插拔 | 支持麦克风/摄像头设备的热插拔检测和自动切换 | 拔出设备时提示并切换至可用设备 |

#### 7.6.4 浏览器兼容性要求

| 浏览器 | 最低版本 | 语音面试支持 | 面对面面试支持 | 说明 |
|--------|----------|------------|--------------|------|
| Chrome | 90+ | 完整支持 | 完整支持 | 推荐浏览器，最佳体验 |
| Edge | 90+ | 完整支持 | 完整支持 | 推荐浏览器 |
| Firefox | 88+ | 完整支持 | 支持（性能略低）| 3D渲染性能可能不如Chrome |
| Safari | 14+ | 支持（TTS音色较少）| 支持（需开启WebGL）| iOS Safari限制较多，建议用App |
| 微信内置浏览器 | 最新版 | 支持 | 不支持 | 面对面模式需跳转外部浏览器 |
| 其他浏览器 | -- | 基础功能 | 不支持 | 提示使用推荐浏览器 |

**兼容性检测方案**：
- 进入面对面模式前进行前置检测：WebGL支持、MediaDevices API、SpeechRecognition API
- 检测不通过时展示兼容性提示页面，明确告知问题原因和解决方案
- 提供"一键检测"功能，用户可提前检测设备兼容性

#### 7.6.5 网络与资源需求

| 需求项 | 要求 | 说明 |
|--------|------|------|
| 最低带宽 | 下载2Mbps / 上传1Mbps | 保障语音和3D资源流畅加载 |
| 推荐带宽 | 下载10Mbps / 上传2Mbps | 最佳体验 |
| 弱网降级 | 网络带宽<1Mbps时提示并建议切换文字模式 | 避免卡顿影响面试体验 |
| 资源预加载 | 3D模型和场景资源支持预加载配置 | 可在面试准备页提前加载，减少等待 |
| 断网恢复 | 网络中断时本地缓存音频数据，恢复后自动续传 | 保障面试连续性 |

## 风险评估与应对（补充）

| 风险类别 | 风险描述 | 影响程度 | 应对措施 |
|----------|----------|----------|----------|
| 技术风险 | 语音识别准确率低（特别是中英文混合、技术术语） | 高 | 接入多家ASR服务商做对比；建立技术术语词库；用户可编辑修正 |
| 技术风险 | TTS语音合成效果不自然，机械感强 | 高 | 选用业界领先的TTS引擎（如Azure TTS、阿里云语音合成）；提供多种音色选择 |
| 技术风险 | Three.js 3D渲染性能不足，低端设备卡顿 | 高 | 提供性能模式开关；模型LOD分级加载；做好帧率监控和自动降级 |
| 技术风险 | 口型同步效果不自然，与语音不同步 | 中 | 基于音素精确映射；增加口型过渡插值；口型库持续优化 |
| 产品风险 | 语音模式用户接受度不高，习惯文字输入 | 中 | AB测试对比；保留文字模式作为备选；收集用户反馈持续优化 |
| 产品风险 | 面对面模式加载慢，用户流失 | 中 | 资源预加载机制；加载过程有进度提示和趣味动画；性能持续优化 |
| 安全风险 | 语音数据隐私问题 | 高 | 语音数据仅用于实时转写，不落库存储（除非用户主动保存）；明确的隐私政策 |
| 安全风险 | 摄像头隐私顾虑 | 中 | 摄像头仅在本地展示，视频流不上传服务器；提供仅音频模式 |

### 新增术语表

| 术语 | 说明 |
|------|------|
| ASR | Automatic Speech Recognition，自动语音识别，将语音转换为文字 |
| TTS | Text-to-Speech，文本转语音，将文字转换为语音播报 |
| VAD | Voice Activity Detection，语音活动检测，检测用户是否在说话 |
| WebRTC | Web Real-Time Communication，网页实时通信技术，用于麦克风采集 |
| WebGL | Web Graphics Library，网页图形库，用于浏览器端3D渲染 |
| Three.js | 基于WebGL的JavaScript 3D库，用于构建3D场景和模型 |
| Blend Shape | 3D模型的变形技术，通过混合多个基础表情实现丰富面部表情 |
| Viseme | 视位，与音素（Phoneme）对应的视觉口型 |
| LOD | Level of Detail，细节层次，根据距离和性能动态调整模型精度 |
| MediaDevices API | 浏览器提供的媒体设备访问接口，用于获取摄像头和麦克风 |
| 口型同步 | Lip Sync，将语音与3D模型的嘴唇动作进行时间对齐的技术 |
| 毛玻璃效果 | Frosted Glass，半透明模糊的背景效果，用于UI面板 |

### 技术选型建议

| 技术领域 | 推荐方案 | 备选方案 | 说明 |
|----------|----------|----------|------|
| 语音识别(ASR) | 阿里云智能语音识别 | 百度语音、腾讯云语音识别 | 支持中文技术术语，实时流式识别 |

| 音频处理 | Web Audio API | -- | 浏览器原生支持，低延迟 |
