# P0 优先级优化开发计划

## 目标

把当前可演示的 AI 语音视频面试系统推进到更稳定的内部试用版本，重点解决 AI 输出不稳定、模型配置写死、面试状态不严谨、采集反馈不清晰、语音/设备降级不足、后台页面割裂等问题。

## 范围

### 1. 后台模型配置

- 新增模型配置数据表，支持多个 OpenAI 兼容模型配置。
- 后台可新增、编辑、启用模型配置。
- AI 服务每次调用读取当前启用配置，不再把模型固定在 `.env`。
- `.env` 仅作为首次启动的默认种子配置。
- API 返回时隐藏完整 API Key，仅展示脱敏值。

### 2. AI 结构化输出稳定性

- 抽出统一 JSON 提取逻辑，支持纯 JSON、Markdown JSON 代码块、前后带解释文字的 JSON。
- 简历解析和评估报告都使用统一提取逻辑。
- 评估字段做归一化：缺失字段补默认值，非法分数归 70，数组/对象字段类型兜底。
- 模型返回非 JSON 时保留默认降级结果，不抛业务异常。

### 3. 面试状态增强

- 面试结束流程增加 `completing` 状态，避免重复点击结束导致重复评估。
- 评估生成完成后进入 `completed`。
- 评估生成异常时进入 `failed`，并生成默认评估，用户可以看到失败原因。
- 发送消息只允许 `ongoing` 状态。

### 4. 爬虫可靠性和反馈

- 采集任务区分 `success / failed / no_data`。
- 手动触发后任务历史能展示错误原因、新增数量、重复数量。
- 单个来源失败不影响调度器运行。

### 5. 语音/设备降级

- 前端增加能力检测：语音识别、语音播报、WebGL、摄像头。
- 不支持语音识别时自动回退文字模式。
- 3D 页面 WebGL 不可用时提示切换文字/语音面试。

### 6. 后台统一左侧菜单

- 新增后台布局组件。
- `/admin/questions`、`/admin/crawl`、`/admin/model-configs` 都使用后台左侧菜单。
- 左侧菜单包含：题库管理、采集管理、模型配置、返回面试系统。

## 实施顺序

1. 写后端模型配置和 AI 输出稳定性的回归测试。
2. 新增模型配置表、Schema、服务和路由。
3. 接入 AIService 运行时模型配置。
4. 重构 JSON 提取和评估归一化。
5. 增强面试结束状态流。
6. 补爬虫任务状态和错误反馈。
7. 新增前端后台布局和模型配置页面。
8. 增加语音/设备能力提示。
9. 构建前后端、重启容器、跑 API 和页面验证。

## 验收标准

- 后台可切换模型，不改 `.env` 也能改变后续 AI 调用模型。
- 模型返回 Markdown/解释文字包裹 JSON 时，简历和评估都能解析。
- 模型返回非 JSON 时，评估接口不报错，返回默认评估。
- 重复点击结束面试不会创建重复评估。
- 后台三个页面都有左侧菜单。
- 采集任务能展示成功、失败和无数据原因。
- 前端构建通过，后端单测通过，Docker 后端健康检查通过。
